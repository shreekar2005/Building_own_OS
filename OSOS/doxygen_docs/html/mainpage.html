OSOS - 32-bit Operating System Documentation

Welcome to the official documentation for OSOS, a 32-bit kernel written from scratch in C++.

This site provides a detailed look at the internal structure of the OSOS kernel, generated automatically by Doxygen. You can use the navigation panel to explore all classes, namespaces, and files, complete with inheritance diagrams and dependency graphs.

Core Features & Modules

The following features are implemented in the kernel. You can find their full documentation by exploring the Namespaces and Classes sections.

  • Custom Kernel Library (libk):

    • basic: Provides kiostream (for printf, screen control) and kmemory (for custom new/delete).
    • driver: Includes the DriverManager and drivers for the kkeyboard and kmouse.
    • essential: Contains the GDT (Global Descriptor Table) setup and kicxxabi for global constructor/destructor support.
    • hardware_communication: Provides kport (for low-level I/O port access) and the InterruptManager (for handling the IDT and ISRs).

  • Global Constructor/Destructor Support: C++ global objects are correctly constructed before kernelMain is called, using essential/kicxxabi.

  • Hardware Initialization:

    • Global Descriptor Table (GDT)
    • Interrupt Descriptor Table (IDT)

  • Interrupt Handling: Manages hardware interrupts from the Keyboard and Mouse via registered ISRs.

Code Structure & Namespaces

The kernel's code is divided into several key namespaces to keep it organized. These namespaces form the basis of the documentation's structure:

  • basic: Core functions like I/O and memory.
  • driver: Device drivers (Keyboard, Mouse).
  • essential: Critical boot-time setup code (GDT, C++ ABI support).
  • hardware_communication: Low-level port I/O and interrupt management.
