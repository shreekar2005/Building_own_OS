OSOS - 32-bit Operating System - Developer Documentation

Welcome to the official documentation for OSOS, a 32-bit kernel written from scratch in C++.

This site provides a detailed look at the internal structure of the OSOS kernel, generated automatically by Doxygen. Use the navigation panel to explore the kernel's structure.

How to Find Details

This documentation is generated directly from the source code comments. The best way to find information is to navigate using the Doxygen-generated pages:

  • To see all classes and structures: Click on the "Classes" tab in the navigation bar. For a graphical view, use the "Class Hierarchy" option.
  • To see all namespaces: Click on the "Namespaces" tab. This is the best way to see the kernel's high-level organization.
  • To explore individual files: Click on the "Files" tab to see a list of all documented source and header files.
  • To understand a specific component (e.g., GDT or Keyboard Driver):
    1. Go to the "Namespaces" tab and select the relevant namespace (like essential or driver).
    2. Click on the class you are interested in (e.g., GDT or KeyboardDriver).
    3. On the class page, you will find a detailed description, a list of public/protected functions, and inheritance diagrams.

My personal rules while developing OSOS (IMP if you are thinking to Contribute)

  • Using #include <> for standard libraries
  • Using #include "" for OSOS specific libraries, e.g. libraries in ./libk_headers/
  • Naming OSOS kernel libraries with prefix 'k', e.g. kiostream.hpp, kmemory.hpp, etc
  • Naming convensions :
    • pointer names : abc_xyz_pqr
    • class/struct names : AbcXyzPqr or ABC_XyzPqr
    • function/variable names : abcXyzPqr
  • all kernel library functions, classes, variables are divided in namespaces :
    • basic : have libraries which contains basic IO, memory management etc.
    • driver : have libraries which are for managing drivers e.g. kkeyboard.hpp
    • essential : have libraries which are must for working e.g. kgdt.hpp
    • hardware_communication : have libraries which deals with hardware e.g. kport.hpp
